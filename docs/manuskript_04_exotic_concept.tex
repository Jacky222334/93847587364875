\section{Concept summary}
\textbf{Core idea:} Define a clinically interpretable construct that goes beyond the standard question ``attachment insecurity $\rightarrow$ more utilization'':

\begin{itemize}
  \item \textbf{Perioperative Utilization Amplification (PUA):} the excess peri-intake care/intervention intensity beyond what would be expected from objective disease burden and care complexity.
  \item \textbf{PCI\textsuperscript{3}: Perioperative Care \& Intervention Intensity Index (primary outcome):} a z-scaled composite index combining short-term utilization intensity and intervention proxies.
\end{itemize}

\section{Operational definitions}
\subsection{Primary index (PCI\textsuperscript{3})}
\texttt{periop\_intensity\_index\_z} is computed as the (equal-weighted) mean of the available PCI\textsuperscript{3} components.

\subsection{Amplification (PUA)}
PUA is implemented as the baseline residual:
\begin{itemize}
  \item Baseline model: \texttt{periop\_intensity\_index\_z \textasciitilde cci\_z + opsev\_z + onco\_z (+ age\_z + sex\_bin)}
  \item \texttt{PUA\_residual = observed - predicted\_baseline}
\end{itemize}

\section{Statistical design (response surface)}
Main model (robust OLS HC3 fallback in this environment):
\begin{verbatim}
periop_intensity_index_z ~ anx_z + avoid_z + anx_z^2 + anx_z:avoid_z + avoid_z^2
                           + cci_z + opsev_z + onco_z (+ age_z + sex_bin)
\end{verbatim}

Surface parameters (report with intervals in the final Bayesian version if desired):
\[
a_1=b_1+b_2,\quad a_2=b_3+b_4+b_5,\quad a_3=b_1-b_2,\quad a_4=b_3-b_4+b_5
\]
